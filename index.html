    Meet

    A comprehensive video conferencing and real-time communication library for web applications, built as part of the Talview webclients monorepo.

    Contents

    This package provides:

    • Video Conferencing: Integration with LiveKit and Twilio for real-time video communication
    • Device Management: Camera and microphone permissions and Devices list management
    • Messaging System: Real-time messaging capabilities between participants
    • State Management: Using Automerge and XState for robust state management
    • State Replay: Replaying past states at specific timestamps using Automerge
    • Apollo Integration: GraphQL client integration for backend communication
    • Diagnostics: Tools for testing audio/video devices and diagnosing connection issues

    Installation

    This package is part of a monorepo and is not published to npm. To use it in your project:

    # From the root of the monorepo
    nx build meet

    Usage

    Basic Initialization

    import Meet from '@packages/meet';

    Meet.init(
      // Callback that receives initialization update
      (state) => {
        console.log('Meet state updated:', state);
      },
      {
        // Channel identification
        channelName: 'my-meeting-room',

        // Either provide an Apollo client
        apolloClient: 'apollo-client-instance',
        // Or provide authentication tokens
        userToken: 'your-user-token',

        // Optional settings
        messageOnly: false,
        deviceType: 'primary',
        recordingConsent: true,
      }
    );

    Diagnostics Usage

    import Meet from '@packages/meet';

    // Initialize diagnostics
    Meet.diagnostics.init(
      // Callback that receives error (if any)
      (error) => {
        if (error) {
          console.error('Diagnostics initialization failed:', error);
        } else {
          console.log('Diagnostics initialized successfully');
        }
      },
      {
        channelName: 'my-meeting-room',

        // Either provide an Apollo client
        apolloClient: 'apollo-client-instance',
        // Or provide authentication tokens
        userToken: 'your-user-token',

        // Optional: specify provider (defaults to LIVEKIT_SELF_HOSTED)
        provider: 'livekit-self-hosted',
      }
    );

    // Test media devices
    const videoTestResult =  Meet.diagnostics.testVideoInputDevice(deviceId);
    const audioInputTestResult =  Meet.diagnostics.testAudioInputDevice(deviceId);
    const audioOutputTestResult =  Meet.diagnostics.testAudioOutputDevice(deviceId);

    State Replay Usage

    import Meet from '@packages/meet';

    async function initStateReplay() {
      const stateReplayInstance = await Meet.stateReplay.init({
        channelName: 'my-meeting-room',
        // Either provide an Apollo client
        apolloClient: 'apollo-client-instance',
        // Or provide authentication tokens
        userToken: 'your-user-token',
      });

      // Use the state replay instance
      stateReplayInstance.seekToTimestamp(1749438873);
    }

    Device Management

    import Meet from '@packages/meet';

    // Get available devices
    const devices = await Meet.deviceManager.getDevices();

    Documentation

    Generate and view the API documentation:

    # Generate documentation
    nx run @packages/meet:docs:generate


    # Serve documentation
    nx run @packages/meet:docs:serve

    The documentation will be available at http://localhost:8080.

    Architecture

    The package is organized into several modules:

    • DeviceManager.ts: Manages media devices (cameras, microphones)
    • initializeMachine.ts: XState machine for initialization flow
    • message/: Real-time messaging system
    • room/: Video room management
    • stateManager/: State management using XState
    • stateReplay/: State replay using Automerge
    • diagnostics/: Tools for testing and diagnosing media devices and connections
    • utils/: Utility functions and helpers
